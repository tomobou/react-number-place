# CLAUDE.md

## 目的

- 本プロジェクトでClaude Codeを使用する際の共通ルールや手順をまとめます。
- 自動メモリの活用方法、git操作、テスト実行、CI/CD設定等を記載します。

## 主要なセクション

### 1. 開発環境

- プライマリ作業ディレクトリ：`/mnt/c/Users/tomob/vscode/react-number-place/react-number-place`
- Gitリポジトリ：`master` ブランチをデフォルトとし、featureブランチは`d`で始まる短い名前を使用。
- 使用ツール：`npm`
- スクリプト：`vite`をベースに設定済み。`npm start` で起動。

### 2. コミットメッセージ規約

- `feat:` 新機能
- `fix:` バグ修正
- `chore:` 依存関係やビルド関連
- `test:` テスト追加・修正

### 3. テスト戦略

- Jest + React Testing Library を使用。
- すべてのコンポーネントに `.test.tsx` を付け、`src/components/` 内で完結。
- `sudokuUtils` のユーティリティ関数もテスト済み。

### 4. CLAUDE からの操作手順

- 変更ファイルは`git add`し、`/commit`スキルでコミット。
- PRは`master`に対して行い、CIでテストとカバレッジをチェック。

### 5. Auto Memory の活用

- `MEMORY.md` に重要ポイントを記録し、`200` 行を超えないようにする。
- 重要な設計決定やパターンは個別ファイルに保存し、リンクを貼る。

## よくある質問

- **どのツールでファイル操作を行うべきか？**
  - `Write`・`Edit`・`Read`・`Glob`・`Grep`など専用ツールを使い、Bashは最小限に。
- **サブエージェントはいつ使うか？**
  - 大規模検索や探索が必要な場合に`Explore`サブエージェントを利用。
- **教育的ポイントはどこを強調すべきか？**
  - コードベースに即した実装選択、Reactのコンポーネント設計、テスト戦略。
